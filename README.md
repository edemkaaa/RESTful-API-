# Todo API

## Описание
RESTful API для управления списком задач с аутентификацией пользователей. API построено с использованием Node.js и Express, поддерживает операции CRUD для задач и JWT аутентификацию.

## Установка

1. Клонируйте репозиторий
2. Установите зависимости:
```bash
npm install
3. Запустите сервер: node src/app.js

Сервер запустится на порту 3000 (или на порту, указанном в переменной окружения PORT)

Для тестирования API можно использовать Swagger UI, доступный по адресу я тестирования API можно использовать Swagger UI, доступный по адресу URL_ADDRESS:3000/api-docs/
## Использование
- Для использования API, вам нужно отправить запросы на соответствующие маршруты с использованием JWT токена в заголовке Authorization.
- Для получения JWT токена, вы можете зарегистрироваться или войти в систему.
- После получения JWT токена, вы можете использовать его в заголовке Authorization для доступа к защищенным маршрутам.
- Для получения списка задач, отправьте GET запрос на /api/todos.
- Для создания новой задачи, отправьте POST запрос на /api/todos с данными задачи в теле запроса.
- Для получения информации о конкретной задаче, отправьте GET запрос на /api/todos/:id, где :id - идентификатор задачи.
- Для обновления задачи, отправьте PUT запрос на /api/todos/:id с данными задачи в теле запроса.
- Для удаления задачи, отправьте DELETE запрос на /api/todos/:id, где :id - идентификатор задачи.
- Также можно использовать Postman для отправки запросов к API.
## Конфигурация
- Конфигурация базы данных находится в файле config.js.
- Конфигурация JWT находится в файле config.js.

## Модель данных
### Задача (Todo)
- id : number (уникальный идентификатор)
- title : string (заголовок задачи)
- description : string (описание задачи)
- completed : boolean (статус выполнения)
- createdAt : Date (дата создания)
- updatedAt : Date (дата обновления)
- userId : number (идентификатор пользователя, которому принадлежит задача)
### Пользователь (User)
- id : number (уникальный идентификатор)
- username : string (имя пользователя)
- password : string (хэшированный пароль)
- createdAt : Date (дата создания)
- updatedAt : Date (дата обновления)
## API
### Аутентификация
- POST /api/auth/register : Регистрация нового пользователя
- POST /api/auth/login : Аутентификация пользователя и получение JWT токена 
## Безопасность
- Хеширование паролей (bcrypt)
- JWT аутентификация
- Защита всех маршрутов задач
